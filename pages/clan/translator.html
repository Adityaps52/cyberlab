Translator

What is translator?

  • A translator is a computer program that converts the soure code into machine understandable form(machine code) and vice versa.
  • In other words, A translator is a program that converts the high level language to low level and vice versa.

    • The code that is written by the programmer is termed as the source code whereas the form of code that a computer can understand is called as the machine code.

  • The primary task of the translator is to convert the high level language into low level whereas it also helps the programmer in detecting the errors in the code.

Types of Translator

Translator has been categorised into 3 types i.e

  1. Assembler
  2. Compiler
  3. Interpreter

Let us discuss each translator in details

  1. Assembler

    2. An assembler is a translator program that converts the assembly level language to machine understandable form. The assembler can be termed as the complier of the assembly level language because it performs the same activity as the compiler does in the high level language.

  2. Compiler

    3. A compiler is a translator program that converts the high level language into the machine understandable form in a single session.. The execution of the compiler in comparision to other high level translator is fast. A complier generates a machine code that runs on a processor with a specific instruction.

  3. Interpreter

    4. Interpreter performs the same activity what a complier does. An interpreter converts translates code from high level to machine level but reads the code line-by-line and executes its immediately. Thus, interpreter are often used in software development tool as a debugger.

Comparison between Compiler and Interpreter

In order to understand the compiler and the interpreter the best method is to understand the difference between the two translator programs.

Compiler Interpreter
Compiler scans whole program in one session Interpreter scans the program line-by-line
As complier scans whole program at once so all the bug encountered while scanning are shown at end. Bugs are displayed at end of each line.(If found)
Compiler is faster than interpreter in terms of execution Interpreter are slower than compiler
Compiler consumes very less time to execute a program Interpreter cosumes more time than compiler
Compiler does not necessiate the use of source code for later execution Interpreter necessiates the use of source code for later execution.
CPU utilisation is more in compiler Interpreter uses the CPU less as compared to compiler
Program execution is very fast Program execution is slower
Error detection is difficult Error detection is easier as bugs are shown at the end of each scanned line.
Compiler consumes more memory Interpreter need less memory as compared to interpreter

Bug and Debugging

A Bug refers to an error, fault or flaw in any computer program or a hardware system. A bug produces unexpected results or causes a system to behave unexpectedly. It is any behavior or result that a program or system gets but it was not designed to do. Computer bugs are due to human error.
Bug or error can be categorised into 3 types i.e

  1. Syntax Error

    2. Syntax error other wise called as the grammatical mistakes that occurs due to incorrect spelling,inapropriate use of puncuation, incorrect labels and so on. A compiler can detect the syntax error only

  2. Logical Error

    3. A Logical error is an error the produces unexpected results and Operate the program incorrectly. No translator can detect the Logical error.
    This error occurs due to the violation of the syntax rules in the particular programming language.

  3. Runtime Error

    4. A runtime error in a program is an error that occurs while program is running after being successfully complied. This type of error is fixed automatically after restarting the application.

Debugging is the process of removing or rectifying the error present in the code process that involves identifying a problem, isolating the source of the problem and then either correcting the problem or determining a way to work around it.
